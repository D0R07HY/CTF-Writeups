# unfinished-file (Forensics, 356pts)

**Flag:** `tjctf{n3v3r_l3t_0ther_p30ple_t0uch_ur_c0mputer}`

## โจทย์

ให้ไฟล์ที่ดูเหมือนเสียหรือดาวน์โหลดไม่สมบูรณ์ แต่ข้อมูลบางส่วนยังคงกู้กลับมาได้

## ขั้นตอน

1. เปิดดู raw bytes หรือ strings ที่ยังเหลืออยู่ในไฟล์
2. ทดลอง XOR ด้วยค่าคงที่ที่น่าสงสัย
3. พบว่าการ XOR ด้วย `0x42` ทำให้ข้อมูลกลับมาอ่านได้
4. ดึงข้อความที่เป็น flag ออกมา

## หลักการ

โจทย์แนวนี้มักไม่ได้ต้องซ่อมไฟล์ทั้งก้อน แค่หาให้ได้ว่าส่วนข้อมูลที่เหลือถูก obfuscate แบบไหน

## สรุป

- ไฟล์เสียไม่ได้แปลว่าข้อมูลข้างในหายหมด
- ลอง `strings`, hex view, และ XOR ค่าคงที่ก่อนเสมอ

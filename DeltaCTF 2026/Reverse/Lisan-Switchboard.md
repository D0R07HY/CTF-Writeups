# Lisan Switchboard (Reverse, 400pts)

**Flag:** `0xV01D{vm_tables_do_not_care_about_prompt_files}`

## โจทย์

ไบนารีใช้ระบบแปลงข้อมูลแบบ custom ที่พึ่งตาราง substitution และ state ภายใน คล้าย mini VM

## ขั้นตอน

1. วิเคราะห์ส่วนที่รับ input และส่วนที่อัปเดต state
2. ดึงตารางหรือข้อมูล lookup ที่ฝังอยู่ในไฟล์ ELF
3. เขียนสคริปต์จำลองการทำงานของตัวตรวจ
4. ย้อน logic กลับจนกู้ input ที่ทำให้เงื่อนไขผ่านได้

## หลักการ

โจทย์ VM หรือ table-driven checker มักแก้ได้ด้วยการย้าย logic ออกมาใส่สคริปต์ แล้วค่อย ๆ ย้อนลำดับการทำงานแทนการอ่าน assembly ทั้งหมดในหัว

## สรุป

- ดึง table ออกมาก่อนแล้วค่อยอ่าน logic
- เมื่อจำลอง state ได้ครบ การกู้ flag จะง่ายขึ้นมาก
